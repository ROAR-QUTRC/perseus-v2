## Fast-LIO

This package has git submodules, which can't be retrieved by fetchFromGitHub
automatically. This has to be overridden from the generated package. Setting
fetchSubmodules to true just uses fetchGit instead, so I've used fetchGit to be
cleaner. Fast-LIO also compiles specifically for c++14, which conflicts with the
rclcpp library, which is written for c++17.

## Lidarslam Licenses

Each of the lidarslam packages contain the license BSD2. While the official
license is formatted as "BSD-2-Clause", the license in nixpkgs.lib.licenses is
formatted as "bsd2" which causes problems when autogenerating the packages. Each
of these needed to be patched exactly the same, so I made a function at the top
of patches.nix which replaces the license string with the proper nixpkgs license
value.

## Lidarslam-msgs

This package has ament-cmake-auto missing from its package.xml, so the
autogenerated nix package also doesn't contain the dependency.

## Ndt-omp-ros2

This package includes pcl/filters/boost.h, which has been removed as it was just
including header files from the Boost package. It also has an older cpp
standard, which is unsupported by pcl. These have been changed to reference all
of the boost headers and cpp version 17 instead.

The autogenerated package also has multiple packages in buildInputs instead of
propagatedBuildInputs. These packages are needed by any package that includes
this package (e.g. lidarslam, graph-based-slam, scanmatcher), so they've been
moved to propagatedBuildInputs.

## Graph based slam

This package has nav_msgs and std_srvs missing from its package.xml, so the
autogenerated nix package also doesn't contain the dependencies.

## Scanmatcher

This package had test inputs (ament-lint-auto, ament-lint-common) marked as
build inputs in its package.xml, so the autogenerated package put them in
propagatedBuildInputs. The packages have been moved to checkInputs.
